# Supabase Setup

This app now supports a manual Supabase sync for profiles and tasks.

## 1) Create tables in Supabase SQL editor

```sql
create table if not exists public.profiles (
  owner_key text primary key,
  display_name text not null default '',
  email text not null default '',
  organization text not null default '',
  updated_at timestamptz not null default now()
);

create table if not exists public.tasks (
  id bigint generated by default as identity primary key,
  owner_key text not null,
  title text not null,
  original_message text not null,
  sender text not null,
  deadline_date text not null,
  deadline_time text not null,
  deadline_timestamp bigint not null,
  priority text not null,
  category text not null,
  reminder_minutes_before integer not null,
  source_app text not null,
  status text not null,
  created_at bigint not null,
  updated_at timestamptz not null default now()
);

create unique index if not exists tasks_owner_source_sender_message_idx
on public.tasks(owner_key, source_app, sender, original_message);
```

## 2) Add app config to gradle.properties

Set these values in your project `gradle.properties` (or your global `~/.gradle/gradle.properties`):

```properties
SUPABASE_URL=https://YOUR_PROJECT_ID.supabase.co
SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY
SUPABASE_TASKS_TABLE=tasks
SUPABASE_PROFILES_TABLE=profiles
```

## 3) RLS policy note (important)

For production, enable auth and write strict RLS policies by user id.
For initial testing only, you can keep RLS off on these tables.

## 4) Use in app

- Open Settings -> Account & Sources
- Save profile/source rules
- Tap `Sync With Supabase`
- Sync status appears below the button

## 5) Current sync behavior

- Pushes all local tasks to Supabase
- Pulls tasks for the current `owner_key` back to the phone
- Uses upsert key: `(owner_key, source_app, sender, original_message)`

This is a practical starter sync and can later be upgraded to full auth-based multi-device real-time sync.
